可持续更新的通配符证书申请

最近尝试用Let's Encrypt来添加通配符证书，发现之前的工具certbot在使用上有许多不方便之处。 比如证书的注册时，Let's Encrypt会要求你配置dns解析来验证，这一点会影响证书的自动化更新， 并且在安装时经常需要处理依赖的问题。最终转向新工具acme.sh，安装简单，使用方便。

安装

curl https://get.acme.sh | sh

提示安装成功后执行 source ~/.bashrc，就可以在命令行使用acme了。

申请证书

申请证书，可以直接执行acme.sh --issue -d example.com -d *.example.com 注意这里我们同时申请了根域名和通配符域名。另外我在上面提到acme的好处之一，可以自动化dns验证： 以阿里云为例，首先需要去阿里云dns控制面板获取api所需要的App_Key和App_Secret，

# 替换成从阿里云dns后台获取的密钥
export Ali_Key="xxxxxxxxxxxxxxxxxxxxxx"
export Ali_Secret="xxxxxxxxxxxxxxxxxxxx"
# 换成自己的域名
acme.sh --issue --dns dns_ali -d example.com -d *.example.com

acme会通过线程等待120秒，申请大概需要2分多钟，成功后证书会存到~/.acme.sh/example.com目录。

启用证书

这里有一点需要注意，默认生成的证书都放在安装目录下: ~/.acme.sh/, 建议不要直接使用这个目录, 这些文件大多是acme内部使用, 而且目录结构也可能会变化。

正确的方式是使用 --installcert 指定目标位置, 然后证书文件会被复制到相应的位置：

acme.sh --installcert \
    --cert-file /etc/nginx/cert.d/$domain/$domain.cer\
    --key-file /etc/nginx/cert.d/$domain/$domain.key\
    --ca-file /etc/nginx/cert.d/$domain/ca.cer\
    --fullchain-file /etc/nginx/cert.d/$domain/fullchain.cer\
    --reloadcmd "sudo service nginx force-reload"

acme做了记忆化，会将上面的操作写入到cron任务中，但是sudo操作需要手动输入密码会比较麻烦，因此我们可以修改一下sudoer文件，让sudo service nginx force-reload不需要输入密码

sudo emacs /etc/sudoers

然后添加一行内容

test(登录名)  ALL=(ALL) NOPASSWD: /usr/sbin/service nginx force-reload

这样，我们只需要在nginx中配置好证书的地址，就可以完成证书的全自动更新了

server {
    server_name example.com;
    listen 443 ssl;
    # ssl on;
    ssl_certificate   /etc/nginx/cert.d/example.com/fullchain.cer;
    ssl_certificate_key  /etc/nginx/cert.d/example.com/example.com.key;

    ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
}

更新

acme在安装的时候往crontab增加了一行每天执行的命令，证书会在过期后自动更新, 你无需任何操作。

完整的脚本

# 替换成从阿里云dns后台获取的密钥
export Ali_Key="xxxxxxxxxxxxxxxxxxxxxx"
export Ali_Secret="xxxxxxxxxxxxxxxxxxxx"

domain="your-domain.com"
mkdir -p /etc/nginx/cert.d/$domain
acme.sh -d "$domain" -d "*.$domain" \
    --issue --dns dns_ali \
    --installcert \
    --cert-file /etc/nginx/cert.d/$domain/$domain.cer\
    --key-file /etc/nginx/cert.d/$domain/$domain.key\
    --ca-file /etc/nginx/cert.d/$domain/ca.cer\
    --fullchain-file /etc/nginx/cert.d/$domain/fullchain.cer\
    --reloadcmd "sudo service nginx force-reload"
# 修改sudoer文件
